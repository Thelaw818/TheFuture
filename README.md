## Local-only upgrade notes

This package is configured as a local-only tracker. Data is saved on the user's browser/device and is not uploaded to any server by default. The login screen is a local privacy screen, not cloud authentication. Users should export JSON backups regularly because clearing browser/site data, using private browsing, or changing devices can remove local records.

Recommended operating procedure:
1. Create a local profile on the device.
2. Enter records normally.
3. Export a full JSON backup after major updates.
4. Store backup files somewhere safe.
5. Import the JSON backup on another device only when intentionally transferring records.
## Local-only Live Protocol Calendar

The Protocol Builder now includes a Live Protocol Calendar. It stays completely local and automatically combines:

- Recurring protocol schedule entries created in Protocol Builder
- Completed protocol checkoffs
- Peptide Log records
- Dose Log records

The calendar reads from browser local storage only. No protocol, dose, injection, or calendar data is uploaded to any server.
